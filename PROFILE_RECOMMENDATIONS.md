# GitHub Profile Recommendations

This file summarises practical recommendations for improving the public GitHub portfolio behind the `jmDarcy` profile. The assessment is based on the current public repositories and the local CV/profile context in `C:\Users\jakob\OneDrive\Pulpit\CV_pliki\Nowe_CV`.

## Recommended Pinned Repositories

Pin these first, in this order:

1. `telematics_insurance_project`  
   Strongest insurance-specific project. It links actuarial pricing, GLM modelling, synthetic telematics data, Kafka/Spark streaming and real-time operational decisions.

2. `markowitz-cardinality-metaheuristics`  
   Strong quantitative finance project with optimisation, metaheuristics, Monte Carlo simulation and GARCH-based scenario generation. Good signal for risk and mathematical modelling.

3. `survival-analysis-heart-failure`  
   Strong statistical modelling project. It shows Kaplan-Meier, Nelson-Aalen, AFT, Bayesian Weibull and Cox PH modelling with diagnostics.

4. `simple-bayesian-models`  
   Good academic modelling repository. It is valuable because it explains Bayesian linear probability and Bayesian logistic models with clear mathematical framing.

5. `data-mining-airline-satisfaction`  
   Good applied machine-learning project with clear model comparison and diagnostics. It is less actuarial, but useful for data analytics and ML roles.

6. `real-time-analytics-labs`  
   Pin only if you want to show breadth in Kafka/Spark/API workflows. It is more lab-oriented than portfolio-oriented, so it should be lower priority than the telematics project.

  The README is strong and mathematical. Add a short "Results summary" near the top so reviewers can understand the empirical comparison before reading the full formulation.
